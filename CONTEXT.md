# Context

Domain language and mental model for this project.

## Glossary

### Core Entities

- **ProcessingTask** ‚Äî A video processing job. Created when a user submits a source video with a template. Tracks status through pending ‚Üí processing ‚Üí completed/failed. Stores the ICE job ID and output URL.
- **VideoTemplate** ‚Äî The current template system. Supports two modes: **simple** (form-based, `is_advanced=False`) and **advanced** (Timeline JSON with placeholders, `is_advanced=True`). Categories: medical, education, general, migrated.
- **TaskStyle** ‚Äî Deprecated legacy template system. Still readable for backward compatibility but no longer created. Uses fixed fields (open/close videos, background images, transitions).
- **DoctorInfo** ‚Äî Doctor profile (name, hospital, department, title). Used to generate text overlays on medical video templates.
- **File** / **Template** ‚Äî Database records for uploaded videos and template materials, storing filename, OSS URL, upload time, and size.

### Template Concepts

- **Simple mode** ‚Äî Form-based template configuration. Defines header video, footer video, subtitle settings, and text overlays. Best for basic video sequences.
- **Advanced mode** ‚Äî Full Timeline JSON configuration with placeholder support. Enables multi-track editing, complex transitions, and dynamic content injection.
- **Timeline** ‚Äî An ICE Timeline JSON structure describing video tracks (`VideoTracks`), subtitle tracks (`SubtitleTracks`), and audio tracks (`AudioTracks`). Each track contains clips with `MediaURL`, `TimelineIn`, `TimelineOut`, and other properties.
- **Placeholder** ‚Äî Tokens in Timeline JSON replaced at processing time. `$main_video` (source video URL), `$mainSubtitleDepart` (hospital + department as vertical text), `$mainSubtitleName` (doctor name + title as vertical text), `$beginingSubtitleTitle` (video title for subtitle track), `$beginingAudioTitle` (video title for TTS audio).
- **TimelineFormatter** ‚Äî Strategy that resolves placeholders into actual values and applies formatting (e.g., vertical Chinese text layout for medical overlays).
- **TemplateConverter** ‚Äî Bidirectional converter between simple and advanced template modes.

### Processing Pipeline

- **TaskProcessor** ‚Äî Background engine (ThreadPoolExecutor) that submits jobs to ICE, polls for completion, and extracts subtitles from finished videos.
- **ICE** (Intelligent Creative Editing) ‚Äî Alibaba Cloud video editing service. The external engine that performs actual video composition, rendering, and ASR.
- **OSS** (Object Storage Service) ‚Äî Alibaba Cloud object storage. All media assets (source videos, templates, processed outputs, SRT files) live here.
- **STS** (Security Token Service) ‚Äî Time-limited credentials for direct client-to-OSS uploads, avoiding server-side file proxying.
- **ASR** (Automatic Speech Recognition) ‚Äî ICE-provided speech-to-text for subtitle extraction from video audio.

### Status Terminology

- **pending** ‚Äî Task created, not yet submitted to ICE.
- **processing** ‚Äî Submitted to ICE, waiting for completion.
- **completed** ‚Äî ICE job finished, output video available.
- **failed** ‚Äî ICE job failed or was cancelled.

### Data Formats

- **Timeline JSON** ‚Äî ICE's native format for describing video edits. Schema-validated before storage.
- **SRT** ‚Äî Subtitle file format. Generated by ASR and stored in OSS under `subtitles/`.
- **Subtitle data** ‚Äî JSON structure stored on ProcessingTask: `{segments: [{index, content, from, to}], srt_file_url, generated_at}`.

### OSS Path Convention

- `uploads/{timestamp}_{filename}` ‚Äî User-uploaded source videos.
- `templates/{timestamp}_{filename}` ‚Äî Template material files.
- `processed_videos/` ‚Äî ICE-produced output videos.
- `subtitles/` ‚Äî ASR-generated SRT files.
