## 修复内容

### ICE API mock不匹配（2个测试失败）

**原因**: `ice_service.py` 实际使用 `submit_asrjob` + `get_smart_handle_job`，但测试mock的是旧接口 `submit_media_producing_job` + `get_media_producing_job`。

**修复**: 更新 `tests/test_subtitle_service.py` 中 `TestICESubtitleJob` 类的3个测试方法：

1. `test_submit_subtitle_job_sends_correct_timeline` → `test_submit_subtitle_job_returns_job_info`
   - mock `client.client.submit_media_producing_job` → `client.client.submit_asrjob`
   - 验证返回 job_id 和 request_id

2. `test_query_subtitle_job_result_returns_segments`
   - mock `client.client.get_media_producing_job` → `client.client.get_smart_handle_job`
   - 模拟 AiResult JSON 格式（非旧版 SubtitleConfig 格式）

3. `test_query_subtitle_job_result_returns_none_on_failure`
   - `get_media_producing_job` → `get_smart_handle_job`


- 字幕内容编辑: 仅更新content字段，from/to时间戳保持不变
- SRT重新生成: 编辑后自动生成新SRT并上传OSS
